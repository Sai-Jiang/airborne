Technologies Used

Descriptions taken from the frameworks' respective pages.

Flask

Flask is a web microframework for Python based on Werkzeug, Jinja2 and good intentions. It is BSD licensed, comes with a built-in development server and debugger, uses RESTful request dispatching, and many more.


MySQL

The MySQL Database powers the most demanding Web, E-commerce and Online Transaction Processing (OLTP) applications. It is a fully integrated transaction-safe, ACID compliant database with full commit, rollback, crash recovery and row level locking capabilities. MySQL delivers the ease of use, scalability, and performance that has made MySQL the world's most popular open source database. Some of the world's most trafficked websites like Facebook, Google, Twitter, Uber, and Booking.com rely on MySQL for their business critical applications.


Dronekit Log Analyzer

DroneKit Log Analyzer (DroneKit-LA) is a powerful open source static analyzer for ArduPilot logs.

It can read and analyze several log formats (*.tlog, *.log, *.bin), output detailed error and warning information in several formats (json, text, summary), and is fast, memory efficient and extensible.


Bootstrap 3

Bootstrap is the most popular HTML, CSS, and JS framework for developing responsive, mobile first projects on the web. Designed for everyone, everywhere. Bootstrap makes front-end web development faster and easier. It's made for folks of all skill levels, devices of all shapes, and projects of all sizes.


AngularJS

Angular is a platform that makes it easy to build applications with the web. Angular combines declarative templates, dependency injection, end to end tooling, and integrated best practices to solve development challenges. Angular empowers developers to build applications that live on the web, mobile, or the desktop

Pymavlink

Pymavlink is the Python implementation of the MAVLink protocol.

Google Maps

Google Maps is a web mapping service developed by Google. It offers satellite imagery, street maps, 360° panoramic views of streets (Street View), real-time traffic conditions (Google Traffic), and route planning for traveling by foot, car, bicycle (in beta), or public transportation. Possible things to improve

There are many things that I didn't manage to do. Here are some of them.

Single Page Application (SPA)

As you may already know, Airborne's frontend and backend are both handled by Flask. Thus, majority of the tasks are done server-side. By turning Airborne into an SPA, some of the work can be taken off the server and it'll (probably) run faster.

RESTFul Backend

If you read the first one, then you've probably seen this coming. Airborne doesn't have a RESTful backend. By having a RESTful backend API, Airborne's functionality can be easily ported to other platforms and applications.

More equipment types

So far, drones are the only equipment included in Airborne. By having more equipment types, Airborne can handle more data for the users.

UI/UX

I'm not good at UI.

And other than those mentioned above, additional features are welcome!
